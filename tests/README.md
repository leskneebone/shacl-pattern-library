# Repository tests

Tests will cover RDF syntax, catalogue-to-file integrity, SHACL outcomes, target isolation, deterministic assembly, dependency locks, normalised validation reports, engine matrices and documentation/translation completeness.

Kurra is the preferred local SHACL runner; `riot --validate` or `rapper` provides a quick syntax check. Engine-specific evidence must record exact engine versions and commands. No engine is treated as the specification oracle.
