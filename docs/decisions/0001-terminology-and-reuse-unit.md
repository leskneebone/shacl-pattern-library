# 0001 — Pattern is the catalogue unit

Status: provisional  
Date: 2026-07-19

Use **pattern** for the public unit of discovery because the reusable knowledge includes intent, constraints, parameters, mappings and evidence. Use **module** for an executable shapes graph, **template** for material requiring binding, and **assembly** for a resolved application graph. A SHACL node/property shape remains a standards-defined technical object, not the catalogue's universal package boundary.

This decision avoids implying that every catalogue entry is safe to import. Revisit it after the experimental patterns are assembled into at least two distinct local models.
